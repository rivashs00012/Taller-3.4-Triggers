Triggers: Automatización en Bases de Datos

Los triggers (disparadores) son objetos de base de datos que se ejecutan automáticamente en respuesta a eventos específicos sobre tablas. En este taller hemos implementado triggers para auditoría completa en dos bases de datos diferentes.

La versión actualizada incluye mejoras en los triggers de eliminación y actualización, así como la corrección de errores detectados en la implementación anterior. Hemos asegurado el manejo adecuado de la integridad referencial y la consistencia de datos.

Triggers implementados:

  • TRG_Products_Update - Auditoría de cambios en precios
  • TRG_Products_Delete - Auditoría de eliminación de productos
  • TRG_Orders_Insert - Auditoría de nuevos pedidos

Base de datos de editorial con libros, autores y ventas. Implementamos auditoría para gestionar el ciclo completo de publicaciones.

Triggers implementados:

  • trg_delete_titles - Auditoría de eliminación de libros
  • trg_update_titles - Auditoría de actualización de precios
  • trg_insert_titles - Auditoría de inserción de libros
  • trg_delete_authors - Auditoría de eliminación de autores
  • trg_delete_sales - Auditoría de eliminación de ventas

Se respetó la integridad referencial eliminando primero en tablas hijas antes de activar los triggers.

Ejemplo de Trigger Actualizado
Trigger para Eliminación de Pedidos (Corregido)

Este trigger se activa después de eliminar un pedido en la tabla Orders y guarda todos los datos del pedido eliminado en la tabla de auditoría Log_DeleteOrders.

-- Creación de tabla de auditoría para eliminación de pedidos
IF OBJECT_ID("Log_DeleteOrders") IS NULL
BEGIN
    CREATE TABLE Log_DeleteOrders (
        Login INT IDENTITY(1,1) PRIMARY KEY,
        OrderID INT,
        CustomerID NCHAR(5),
        EmployeeID INT,
        OrderDate DATETIME,
        RequiredDate DATETIME,
        ShippedDate DATETIME,
        DeleteDate DATETIME
    );
END
GO

-- Creación del trigger AFTER DELETE
CREATE OR ALTER TRIGGER TRG_Orders_Delete
ON Orders
AFTER DELETE
AS
BEGIN
    INSERT INTO Log_DeleteOrders
    (OrderID, CustomerID, EmployeeID, OrderDate, RequiredDate, ShippedDate, DeleteDate)
    SELECT
    d.OrderID, d.CustomerID, d.EmployeeID, d.OrderDate,
    d.RequiredDate, d.ShippedDate, GETDATE()
    FROM deleted d;

    PRINT 'Trigger DELETE en Orders ejecutado.';
END;
GO
Lecciones Aprendidas
Hallazgos y Soluciones

Durante la implementación y corrección de los triggers, identificamos y solucionamos los siguientes desafíos:

  • Integridad referencial: Los triggers AFTER DELETE fallan si existen restricciones FK. Solución: Eliminar primero registros hijos.
  • Tipos de datos: Errores por tipos incompatibles (NCHAR(S)). Solución: Especificar tamaño correcto (NCHAR(5)).
  • Tablas inserted/deleted: Entender el scope y contenido de estas tablas virtuales es crucial.
  • Performance: Los triggers afectan el rendimiento. Solución: Mantener lógica simple y eficiente.
  • Transacciones implícitas: Los triggers se ejecutan dentro de la transacción de la operación que los activa.
